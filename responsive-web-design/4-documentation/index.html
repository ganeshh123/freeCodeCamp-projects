JavaScript

Introduction

JavaScript is a cross-platform, object-oriented scripting language. It is a small and lightweight language. Inside a host environment (for example, a web browser), JavaScript can be connected to the objects of its environment to provide programmatic control over them.

JavaScript contains a standard library of objects, such as Array, Date, and Math, and a core set of language elements such as operators, control structures, and statements.

Uses

It can interact with user behaviour, or browser driven events. These are some of its uses:

  • Form Data Validation
  • Dynamic Style/Presentation Effects
  • Dynamic Content Creation/Inclusion
  • Data Acquisition
  • Browser Window Management

There are 2 ways in which JavaScript can be inserted and executed in a webpage:

  1. JavaScript can be directly written into the HTML file between <script></script> tags (current document only
  2. External JavaScript files can be imported using <script src="fileName.JS"></script> in the head section

JavaScript and Java

JavaScript and Java are similar in some ways but fundamentally different in some others. The JavaScript language resembles Java but does not have Java's static typing and strong type checking. JavaScript follows most Java expression syntax, naming conventions and basic control-flow constructs which was the reason why it was renamed from LiveScript to JavaScript.

In contrast to Java's compile-time system of classes built by declarations, JavaScript supports a runtime system based on a small number of data types representing numeric, Boolean, and string values. JavaScript has a prototype-based object model instead of the more common class-based object model. The prototype-based model provides dynamic inheritance; that is, what is inherited can vary for individual objects. JavaScript also supports functions without any special declarative requirements. Functions can be properties of objects, executing as loosely typed methods.

JavaScript is a very free-form language compared to Java. You do not have to declare all variables, classes, and methods. You do not have to be concerned with whether methods are public, private, or protected, and you do not have to implement interfaces. Variables, parameters, and function return types are not explicitly typed.

Syntax
  • JavaScript uses C-style syntax
  • Primitive types, such as Strings, ints, and booleans can be loosely typed using the keyword var: var variableName = value;
  • Code statements are ended with a semi-colon;
  • Curly braces {} are used to group together sections of code
Functions

Functions (known as methods in other languages are ways of grouping together statements for logican execution and flow control:

  • They are defined using the key word function
  • They have a name, can take in arguments, and can return variables
  • Functions typically have the following syntax: function functionName(arg1, arg2, arg3){ code }
  • Function syntax does not require semi-colons, only the code inside
  • The function is called in the following format functionName(arg1, arg2), where the arguments are specified
  • The code inside the curly braces is then executed
